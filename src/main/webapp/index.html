Modeling the Knowledge Base

No Brain, a modelagem é a representação gráfica do conhecimento, seus relacionamentos com outros conhecimentos (pais e filhos) e os tópicos nos quais o mesmo se divide, tudo isso através de um mapa mental. A modelagem faz parte do processo de criação da base de conhecimento propriamente dita e está sincronizada com a base real do robô.

Em um cérebro real, apenas o conhecimento que está sendo utilizado (usar o nome científico disso!) é que fica na “área de trabalho (usar o nome científico disso!)”, enquanto todo o resto fica guardado no “hd” (usar o nome científico disso!). Isto acontece por vários motivos, mas principalmente para não sobrecarregar a mente (ou seria psiquê? usar o nome científico disso) evitando o colapso do sistema cognitivo, ao mesmo tempo em que viabiliza a máxima performance para conclusão do raciocínio já que ele está focando apenas no que interessa. Algo semelhante ocorre no Brain: apenas os conhecimentos relevantes são exibidos, para que a tela não fique cansativa por exibir muitas informações e para que o botmaster possa focar naquilo que realmente interessa. Os conhecimentos são exibidos ou ocultados conforme sua relevância para o conhecimento selecionado.

Modelar a base de conhecimento de um chatbot não é exatamente uma tarefa trivial, pois existe um conjunto de variáveis que afetam diretamente a efetividade de uma boa interação humano-robô, como por exemplo a riqueza do vocabulário de ambos, a clareza e objetividade das mensagens de ambos, a especialização ou a generalização esperada por um usuário sobre um determinado assunto, etc. Além disso determinados assuntos podem ser muito complexos ou muito grandes para serem tratados por um robô.

Pensando nisso o Brain foi projetado para facilitar esta tarefa levando o “dividir para conquistar” muito a sério. Dividindo o conhecimento ele pode ser modelado com finíssima granularidade, sendo reduzido a vários pequenos conhecimentos onde cada um é muito específico, pois foca apenas em um determinado aspecto de um conhecimento maior. Algumas das vantagens desta abordagem é que tem uma visão mais ampla sobre o conhecimento e fica muito mais fácil definir todos os tópicos cobertos por um conhecimento. O resultado dessa abordagem é uma base com conhecimentos muito mais sólidos e completos.

Mas o mais importante para que a base seja bem modelada é que o botmaster domine cada conhecimento que deseja modelar. É necessário que o botmaster já tenha muito claro em sua mente o que deseja representar através do mapa mental do Brain.

Em resumo, é necessário:
1. Definir um público-alvo
2. Definir os temas de interesse para este público
3. Definir o escopo dos temas (os conhecimentos a serem modelados)
4. Para cada item do escopo (cada conhecimento):
1. Entender bem o conhecimento que será adicionado
2. Definir os relacionamentos (ascendentes e descendentes)
3. Definir os tópicos

Topic

O Tópico é um grupo de ensinamentos, uma divisão interna de um Conhecimento, e é específico apenas do Conhecimento ao qual pertence.

Teaching

O ensinamento é a menor unidade de conhecimento passada para o robô, é a instrução propriamente dita do que ele deve fazer conforme o texto digitado pelo usuário. Ou seja, são os ensinamentos que realmente preparam o robô para lidar com as situações criadas pelos usuários.

O ensinamento é específico de apenas um Tópico, por isso ele só faz sentido para aquele Tópico ao qual pertence.

Normalization

É o processo de padronização do texto digitado pelo usuário, adequando este texto a determinadas normas de escrita e gramática utilizadas pelo robô. A normalização atua basicamente nos seguintes pontos:

Sinônimos. Exemplo: “visualizar” → “ver”.
Abreviações/gírias: Exemplo: “vc” → “você”.
Gênero. Exemplo: “Obrigada” → “Obrigado”.
Correções ortográficas. Exemplo: “voce” → “você”.
Correções gramaticais. Exemplo: “Oi como está vc” → “Oi. Como está você?”
